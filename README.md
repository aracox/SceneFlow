# SceneFlow

**Camera-to-Map Visual Intelligence Platform** — frontend-only prototype.

SceneFlow converts camera-detected real-world entities (vehicles, shuttle buses, people, pets, boats, floating waste, incidents) into a live map-based digital twin. This prototype runs entirely in the browser with mock data: **no backend, no real database, no camera streams, no external APIs**.

Pilot site: **Thailand Digital Valley Pilot**, an ~800 m × 800 m mock area centered at `13.805567987114605, 100.57466669475343`.

## Run it

```bash
npm install
npm run dev
```

Then open the printed URL (default `http://localhost:5173`). Designed for 1440×900 and larger screens.

## What it demonstrates

- **Live map dashboard** (MapLibre GL JS with a clean custom GeoJSON basemap — no tile server required)
- **Moving entities constrained to geometry**: vehicles follow road lanes, the shuttle follows its loop route, people and pets follow pedestrian/park paths, boats follow the waterway, floating waste drifts with the water flow. No random movement anywhere — positions are generated by distance-along-LineString.
- **Complete mock database layer** (`src/data/` + `src/services/mockSceneStore.ts`): 26 entities, 19 paths, 12 zones, 8 cameras, 30 minutes of movement points at 1 point/second/entity (generated in memory at startup), derived detection events, and seeded movement clips.
- **Live mode**: simulated clock advances, markers move, KPI cards and the events feed update.
- **Replay mode**: scrub the timeline or play a clip; positions are interpolated between movement points for smooth animation at 1x / 2x / 4x / 8x.
- **Save 5-minute clip**: select an entity, click *Save 5-minute clip* — a `MovementClip` covering the last 5 minutes is stored in frontend state, appears in the clip list, and replays on click (surrounding entities are dimmed).
- **Tracking states**: marker opacity reflects detection confidence; entities outside camera coverage show as `predicted` with a dashed outline.

## How to use it

1. The app opens in **live mode** with the mock clock at "now" (20 minutes of history is available behind it).
2. Click any marker to select it — the right panel shows its state, source camera, confidence, path and attributes, and an orange trail follows it.
3. Drag the timeline slider backwards to enter **replay mode** over the full 30-minute window; *Back to Live* returns to the live clock.
4. With an entity selected, click **Save 5-minute clip** (optionally typing a reason). The clip appears in the bottom-right list; click it to replay it.
5. Use the sidebar checkboxes to toggle map layers (entity types, cameras, zones, paths/lanes, incidents, trails).

## Project structure

```
src/
  app/App.tsx                     # Layout + simulation clock (rAF)
  components/
    layout/                       # Header, Sidebar, KpiCards
    map/                          # SceneMap, EntityMarker, CameraMarker, TrailLayer
    panels/                       # EntityDetail, CameraFeed, Events, MovementClips
    timeline/TimelineControl.tsx  # Live/replay transport + scrubber + clip saving
  data/                           # Mock database: entities, paths, zones, cameras,
                                  # movement points, clips, sim time window
  services/
    geometryUtils.ts              # offsetCoordinate, interpolateAlongLine, headings,
                                  # movement-point generation, point-in-polygon, ...
    replayEngine.ts               # Time-based interpolation + clip summaries
    mockSceneStore.ts             # Frontend-only mock database / query API
  store/sceneStore.ts             # Zustand UI state (mode, time, selection, layers, clips)
  types/scene.ts                  # Entity, MovementPoint, PathGeometry, Zone, Camera,
                                  # MovementClip, render-state types
  styles/index.css                # Tailwind + marker/popup styles
```

## Notes

- All geospatial mock data is generated relative to `MAP_CENTER` via `offsetCoordinate(center, metersEast, metersNorth)` — coordinates stay within lat 13.8020–13.8095 / lng 100.5710–100.5785, in WGS84 `[lng, lat]` order.
- The 30-minute mock window is anchored to the real clock at startup; the live clock wraps around the window if left running past the end.
- Mock data is deterministic per session (seeded PRNG per entity), so speeds and confidence values are stable while you explore.
